
Task 13.4, **Mandatory Source Summary Statement**, requires the report to include a transparent assessment of the quality, credibility, and significant gaps in the underlying source base immediately following the BLUF. This statement enables the decision-maker to calibrate their trust in the analytical conclusions based on the evidential foundation.

The detailed procedural breakdown of this task is as follows:

### 1. Source Base Overview
The statement provides a high-level summary of the sources that underpin the report's conclusions.
*   **Source Diversity:** How many independent sources contributed to the analysis? Were they distributed across multiple tiers (Tier 1 government records, Tier 2 commercial databases, Tier 3 social media, Tier 4 commentary)?
*   **Source Type Distribution:** What proportion of the evidence came from technical sources (system logs, financial records) versus human sources (witness statements, interview transcripts) versus open sources (OSINT)? Technical sources generally carry higher reliability for factual claims.
*   **Admiralty Grade Distribution:** An aggregate summary of the source reliability and information credibility ratings across the evidence base. A report built primarily on A1/B2 sources is more defensible than one relying heavily on F5/F6 sources.

### 2. Significant Gaps and Limitations
The statement explicitly identifies what the source base does not cover.
*   **Missing Source Types:** If entire categories of evidence are absent (e.g., no financial records were accessible, no human sources were available), this limitation is stated.
*   **Jurisdictional Gaps:** If certain jurisdictions did not provide records or were not searchable, the geographic limitations are documented.
*   **Temporal Gaps:** If the source base covers only a portion of the relevant time period (e.g., financial records available only from 2020 onward for an entity incorporated in 2015), the temporal limitation is stated.
*   **Collection Constraints:** If collection was limited by legal restrictions (e.g., GDPR constraints on personal data processing), resource limitations, or access barriers, these constraints are disclosed.

### 3. Single-Source Dependencies
If any critical conclusion depends on evidence from a single source, this dependency is explicitly flagged.
*   **Identification:** The statement lists any analytical judgment that rests primarily on one source, citing the source's Admiralty grade and the judgment it supports.
*   **Risk Assessment:** Single-source dependencies represent analytical fragility — if that source is discredited, the dependent conclusion collapses. The statement assesses this risk.

### 4. Positioning in the Report
The Source Summary Statement is not an appendix — it occupies a prominent position immediately after the BLUF.
*   **Visibility Rule:** The statement must be read before the detailed analysis. This ensures the decision-maker approaches the findings with an informed understanding of the evidence's limitations, rather than discovering caveats buried in footnotes.
*   **No Minimisation:** The statement must present limitations honestly, without downplaying gaps or inflating the quality of the source base. The integrity of the briefing depends on the transparency of this section.
