Task 10.2, **Quantitative Social Network Analysis (SNA)**, applies mathematical algorithms to the edge list (Task 10.1) before any visual interpretation is attempted. The purpose is to identify structurally significant nodes — the hubs, gatekeepers, and bridges — using objective metrics rather than visual impression, which is susceptible to layout bias and the "hairball" effect.

The detailed procedural breakdown of this task is as follows:

### 1. Degree Centrality (Identifying Hubs)
Degree centrality measures how many direct connections a node has. It is the simplest and most intuitive centrality metric.
*   **Calculation:** For each node, count the number of edges connected to it. In a directed graph, this is further divided into in-degree (incoming connections) and out-degree (outgoing connections).
*   **Analytical Significance:** Nodes with high degree centrality are the most connected entities in the network — the people, organisations, or accounts with the greatest number of documented relationships. In criminal network analysis, high-degree nodes often represent leadership figures, key facilitators, or central communication hubs.
*   **Limitation:** Degree centrality measures quantity, not quality. A node may have many connections to peripheral entities while lacking connections to the most analytically significant parts of the network. It must be interpreted alongside other metrics.

### 2. Betweenness Centrality (Identifying Bridges and Gatekeepers)
Betweenness centrality measures how frequently a node lies on the shortest path between other pairs of nodes. It identifies structural chokepoints.
*   **Calculation:** For each node, count the number of shortest paths between all other node pairs that pass through it. Nodes with high betweenness centrality control information flow or resource movement between otherwise disconnected groups.
*   **Analytical Significance:** In investigative contexts, high-betweenness nodes are often the most strategically important targets. A mid-level facilitator connecting two criminal cells may have low degree centrality (few connections) but extremely high betweenness centrality (all communication between the cells flows through them). Removing or compromising this node fragments the network.
*   **Gatekeeper Identification:** Nodes with high betweenness but low degree are classic "bridge" figures — they connect clusters that would otherwise be isolated. These are frequently the most vulnerable points in a network and the highest-priority targets for disruption or intelligence collection.

### 3. Closeness Centrality (Identifying Efficient Communicators)
Closeness centrality measures how close a node is to all other nodes in the network, based on the average shortest path length.
*   **Analytical Significance:** Nodes with high closeness centrality can reach any other node in the network through fewer intermediaries. In intelligence contexts, these nodes can disseminate or receive information faster than others — they are the most efficient communicators in the structure.

### 4. Community Detection and Clustering
Beyond individual node metrics, SNA algorithms identify structural groupings within the network.
*   **Cluster Identification:** Algorithms (e.g., Louvain modularity, Girvan-Newman) partition the network into communities — groups of nodes that are more densely connected to each other than to the rest of the network. Each cluster may represent a distinct cell, business unit, family group, or operational team.
*   **Inter-Cluster Bridges:** The nodes connecting different clusters are identified and cross-referenced with the betweenness centrality results. Nodes that appear as both high-betweenness and inter-cluster bridges are the network's critical structural vulnerabilities.
*   **Temporal Evolution:** If the edge list includes date ranges (from Task 10.1), SNA can be run across different time slices to show how the network's structure changes over time — clusters forming, dissolving, or merging.

### 5. Tooling and Output
SNA calculations are performed using dedicated tools:
*   **NetworkX (Python):** The standard open-source library for graph algorithms, supporting all centrality measures, community detection, and path analysis. Outputs can be exported as data tables or fed into pyvis for interactive visualisation.
*   **Gephi:** Open-source desktop application for large-scale graph visualisation with built-in SNA metrics (degree, betweenness, closeness, eigenvector centrality, modularity).
*   **IBM i2 Analyst's Notebook:** Enterprise tool with integrated SNA functions designed specifically for law enforcement analysts.
*   **Output Format:** SNA results are presented as a metrics table listing each entity, its centrality scores, its cluster membership, and its structural role (hub, bridge, peripheral). This table accompanies the visual output from Task 10.3.
