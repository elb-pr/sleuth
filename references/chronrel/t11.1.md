Task 11.1, **Content Verification**, applies the same digital verification techniques introduced during collection (Task 3.3b) at a deeper analytical level during Phase 4. Where Task 3.3b performed initial vetting to grade incoming intelligence, Task 11.1 deploys the **Content Verification Report** as a structured analytical product that authenticates the provenance of key digital assets and grounds them in the investigation's spatial and temporal framework.

The detailed procedural breakdown of this task is as follows:

### 1. Selection of Assets for Deep Verification
Not every piece of digital media warrants the full verification treatment. The analyst prioritises assets based on their investigative significance.
*   **Diagnostic Assets:** Media that, if authenticated, would significantly alter the ACH matrix in Phase 5 — placing a subject at a specific location at a specific time, or corroborating a key event.
*   **Disputed Assets:** Media whose provenance has been challenged by conflicting evidence, or which originated from low-reliability sources (Admiralty grade "F" on the source axis).
*   **High-Impact Assets:** Media that will be presented in the final briefing (Phase 6) or submitted as evidence in legal proceedings. These require the highest standard of verification.

### 2. Multi-Step Authentication Process
Each selected asset undergoes a structured verification pipeline:
*   **Reverse Image/Video Search (Provenance Check):** The media is submitted to reverse search engines (Google Reverse Image Search, TinEye, Yandex Images) to identify any prior appearances. If the media predates the claimed context, it has been repurposed. If no prior appearances exist, this supports (but does not confirm) its originality.
*   **Geolocation (Spatial Anchoring):** Visual cues within the media — architecture, signage, vegetation, terrain, vehicle types, infrastructure — are matched against satellite imagery (Google Earth, Sentinel Hub) and street-level photography (Google Street View, Mapillary) to establish the precise location depicted. The analyst records the coordinates, the visual cues used, and the confidence level of the match.
*   **Chronolocation (Temporal Anchoring):** The time of capture is independently estimated through: shadow analysis using solar position tools (SunCalc, pysolar) combined with the geolocated coordinates; weather cross-referencing against historical records (Open-Meteo, Weather Underground) for the geolocated area; vegetation and seasonal analysis for rural settings; and visible temporal markers (newspaper front pages, event posters, construction progress).
*   **Metadata Verification:** EXIF data (if present) is extracted and compared against the geolocation and chronolocation findings. Consistency between embedded GPS coordinates and visual geolocation strengthens confidence. Inconsistencies are flagged.

### 3. Manipulation Detection
For high-stakes assets, the analyst assesses whether the media has been digitally altered.
*   **Error Level Analysis (ELA):** Detects composite imagery by analysing compression artefact inconsistencies across the image. Regions that have been pasted or edited display different error levels from the original capture.
*   **Cloning Detection:** Identifies duplicated regions within an image that may indicate objects have been added or removed.
*   **Metadata Inconsistencies:** EXIF data showing a camera model inconsistent with the image resolution or format, or editing software signatures in the metadata, suggests post-capture manipulation.

### 4. Content Verification Report Output
The results of the full verification pipeline are compiled into a structured **Content Verification Report** that becomes part of the permanent case file.
*   **Verification Status:** Each asset is classified as Verified, Partially Verified, Unverified, or Falsified — the same scheme used in Task 3.3b but now applied at analytical depth.
*   **Spatial Coordinates:** The geolocated coordinates and confidence level.
*   **Temporal Estimate:** The chronolocated date/time range and the methods used to derive it.
*   **Admiralty Grade Update:** The verification outcome is fed back to the Source Register, potentially upgrading a source's credibility rating (e.g., an anonymous account posting a verified geolocated video moves from "F6" to "F1").
*   **Evidentiary Linkage:** The report is linked to the corresponding Evidence Register entry (hash-verified via Task 5.1) and to any Entity Register records or master chronology entries that reference the asset.
