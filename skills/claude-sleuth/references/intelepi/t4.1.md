# t4.1 — Enquiry Recording and Assignment

**Template:** `templates/database/task-log.md`

## Procedure

1. YOU MUST log every investigative action in `templates/database/task-log.md` at the moment it is initiated — not upon completion. Retroactive logging is FORBIDDEN.

2. You are REQUIRED to include the following fields in every entry:
   - Unique reference number (e.g., ACT-001)
   - UTC-normalised timestamp
   - Assigned investigator or tool
   - Specific, answerable task description (exact source, specific information sought, expected output format)
   - Linked IR reference from `research/investigation-strategy.md`
   - Priority (High / Medium / Low)
   - Deadline
   - Status (Pending / In Progress / Complete / Cancelled)

3. It is CRITICAL that every action is linked to a documented IR. Actions that cannot be linked indicate potential scope creep — YOU MUST justify them through a Policy Deviation entry (t4.3) before proceeding.

4. ALWAYS ensure only authorised personnel (lead investigator or supervisor) create and assign new actions. Unsanctioned collection is FORBIDDEN.

5. YOU MUST confirm receipt of every tasking with the assigned investigator — creating a two-way accountability record. A task with no confirmed receipt is not operationally active.

6. If an action reveals information that significantly alters scope or direction, YOU MUST escalate to the tasking authority rather than independently expanding the enquiry. The escalation is itself REQUIRED to be logged as a new action.

7. It is FORBIDDEN to mark any action Complete without supervisory review of the outcome.
