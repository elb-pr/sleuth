# t5.2 — Metadata Preservation and Web Archiving

**Tool:** `evidence_preservation.py`, `content_archiver.py`

## Procedure

1. **EXIF Extraction:** YOU MUST extract EXIF metadata from digital photographs and video using ExifTool or PyExifTool. You are REQUIRED to export to JSON or CSV and link to the corresponding Evidence Register entry. GPS coordinates and timestamps MUST be cross-referenced against t3.3b geolocation and chronolocation findings.

2. **Social Media Content:** It is CRITICAL that you understand: absence of EXIF data is EXPECTED behaviour on major platforms — it is NOT an indicator of tampering. ALWAYS record "EXIF stripped by platform" as a factual observation. YOU MUST capture platform-specific metadata (upload timestamps, account identifiers, engagement metrics) as a separate data layer and seek the original uncompressed file where possible.

3. **Web Page Archiving:** YOU MUST capture the complete state of every web page consulted as evidence using the following tools:
   - **Hunchly** for automatic URL tracking, timestamped screenshots, and SHA-256 hashing during browser sessions.
   - **ArchiveBox** for multi-format local capture (WARC, HTML, PDF, PNG).
   - **Playwright** for full-page screenshots with visible URL bar and timestamp.
   - You are REQUIRED to submit every URL to the Wayback Machine via `waybackpy` to create an independent third-party timestamped archive.

4. ALWAYS record chain-of-custody metadata for every capture: collector identity, UTC timestamp, collection method, network context (IP/VPN exit node where relevant), and Evidence Register storage location linked to the SHA-256 hash from t5.1.

5. **Web cache status (current as of 2026):** YOU MUST note that Google Cache was discontinued February 2024 and Bing Cache December 2024. The Wayback Machine is now the primary public cache alternative. Archive.today may be used with caution but is FORBIDDEN as the sole evidentiary archive.
