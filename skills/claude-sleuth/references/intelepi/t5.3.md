# t5.3 — Maintaining the Source Register

**Template:** `research/source-grading.md`

## Procedure

1. ALWAYS create a dedicated entry in `research/source-grading.md` for every source consulted during the investigation. Sources are REQUIRED to remain distinct — merging or de-duplicating source entries is FORBIDDEN.

2. You are REQUIRED to include the following fields in every source entry:
   - Unique Source Identifier (e.g., SRC-001)
   - Source Type: HUMINT / TECHINT / OSINT / Liaison
   - Source Description: sufficient detail for a third party to locate and re-query the source
   - Admiralty Reliability Rating (A–F) from t3.1b with justification
   - Track Record Summary for recurring sources
   - Potential Motivation Assessment: what does the source gain from providing this information?
   - Access Assessment: can the source plausibly know what they claim to know?

3. It is CRITICAL that bidirectional links are maintained: every Entity Register record MUST reference the Source Identifier(s) that contributed each data point, and every Source Register entry MUST list which Entity Register records it has informed. Orphan records (no source) and orphan sources (no downstream records) are integrity failures and MUST be resolved immediately.

4. YOU MUST cascade-reassess all records relying on a source if it is later discredited. Use the bidirectional links to identify every affected record — failure to cascade is a complete systematic failure.

5. ALWAYS apply UK 5×5×5 handling codes. Sensitive source identities MUST be separated from the main case file for dissemination (t14.2).

6. YOU MUST track source lifecycle: Date First Assessed, Last Reviewed, Status (Active / Dormant / Discontinued / Discredited). Stale records not reviewed within the defined threshold MUST be flagged for reassessment before their data is used in any final analytical product.
