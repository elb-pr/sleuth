# t12.4 — Addressing Deception and Gaps

**Template:** `analysis/ach.md`, `research/case-decision-log.md`

## Procedure

### Deception Analysis

1. If one hypothesis appears overwhelmingly supported (only CC/C ratings, no inconsistencies), YOU MUST treat this as a **red flag** requiring suspicion, not confidence. A "too perfect" evidence base MUST be treated as a potential indicator of fabrication — treating it as confirmation is FORBIDDEN.

2. YOU MUST review source motivation (Source Register, t5.3) for every evidence item strongly supporting the leading hypothesis. It is CRITICAL that you assess: does the source have an incentive to promote a specific conclusion? Could they fabricate this evidence?

3. YOU MUST check for circular corroboration: trace apparently independent confirming sources back to their origins. If multiple sources all derive from a single document or statement (identified in t3.2b), the corroboration is illusory and MUST be flagged immediately.

4. You are REQUIRED to review Content Verification Reports (t3.3b) for all digital evidence: Error Level Analysis anomalies, metadata inconsistencies, geolocation/chronolocation mismatches.

5. If deception is suspected, YOU MUST:
   - **Red Team:** Construct the strongest possible case for deliberate fabrication. YOU MUST assess: what evidence would need to be planted? Is it technically feasible? Does the suspected deceiver have access, capability, and motivation?
   - **Provenance Audit:** Trace chain of custody for each piece of supporting evidence. You are REQUIRED to document any breaks where alteration could have occurred.
   - **Re-Score Under Deception Assumption:** Create a variant matrix treating suspected fabricated evidence as II against the hypothesis it originally supported. If the conclusion changes, YOU MUST caveat the original conclusion as fragile.

### Gap Re-Evaluation

6. For each significant gap from t9.2 and the ICP (t2.2), YOU MUST ask: "If this gap were filled, could the new information change the inconsistency score of the leading hypothesis?" If yes: you are REQUIRED to classify it as **conclusion-critical** and escalate for urgent collection.

7. ALWAYS identify asymmetric gaps: gaps that could only add inconsistencies to the current leading hypothesis. YOU MUST acknowledge that the conclusion is conditionally dependent on the gap remaining unfilled.

### Documentation

8. You are REQUIRED to produce a formal annex to the ACH analysis containing all four of the following — omitting any component is FORBIDDEN:
   - **Deception Probability Assessment** — in ICD 203 probability language.
   - **Vulnerable Evidence Register** — items identified as potentially compromised, with specific concerns.
   - **Gap Impact Register** — unfilled gaps with assessed potential to alter the conclusion, prioritised by diagnostic value.
   - **Integrity Statement** — analyst confirmation that deception and gap risks have been formally considered, and whether the conclusion should be caveated accordingly in the Final Briefing.
