# t12.1 — Hypothesis Generation and Matrix Setup

**Template:** `analysis/ach.md`

## Procedure

1. YOU MUST brainstorm all plausible explanations for the investigative problem WITHOUT premature filtering. Apply two structural constraints without exception:
   - **Mutual Exclusivity:** Each hypothesis MUST be distinct — if Hypothesis A is true, Hypothesis B cannot also be true.
   - **Collective Exhaustiveness:** The set MUST cover all reasonable possibilities. A MINIMUM of three hypotheses is REQUIRED — fewer is a satisficing failure and is FORBIDDEN.

2. It is CRITICAL that you include at least one **Devil's Advocate Hypothesis** that directly contradicts the analyst's initial instinct or the prevailing institutional assumption. Its absence is a confirmation bias failure and is FORBIDDEN.

3. YOU MUST assemble the Evidence List (Y-axis) from the full Phase 1–4 output:
   - Entity Register records and relationship data (Phase 3)
   - Chronological Matrix findings — temporal patterns, sequences, and gaps (t9.1–9.3)
   - Network Architecture outputs — centrality scores, cluster structures, two-mode projections (t10.1–10.4)
   - Content Verification Reports — authenticated spatial and temporal evidence
   - Subject and Corporate Profiles — risk indicators, ownership structures, financial anomalies (t8.1)

4. It is CRITICAL that you apply the Inclusion Criterion: include only evidence with **diagnostic potential** — evidence that could, in principle, differentiate between at least two hypotheses. Evidence equally consistent with all hypotheses MUST be excluded.

5. YOU MUST decompose compound evidence items into discrete single facts. "Entity A has links to multiple shell companies AND unusual financial patterns" MUST be split into two separate rows. Compound evidence items are FORBIDDEN in the matrix.

6. You are REQUIRED to list all **Assumptions** separately from facts, explicitly labelled. Each assumption MUST be flagged for sensitivity testing in t12.3.

7. ALWAYS attach to each Y-axis evidence item: its Admiralty grade (t3.1b) and Source Register reference (t5.3). This metadata is REQUIRED for the Sensitivity Analysis — an evidence item without metadata is FORBIDDEN.

8. YOU MUST conduct an Empty Matrix Review before scoring begins: are hypotheses genuinely mutually exclusive? Is the evidence list comprehensive? Are all assumptions labelled? Is each item granular enough? Proceeding to t12.2 without completing this review is FORBIDDEN.
