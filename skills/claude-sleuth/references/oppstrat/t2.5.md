# t2.5 — Establishing the Operational Ontology

**Template:** `research/investigation-strategy.md`

## Procedure

1. It is CRITICAL that you define the operational ontology before any data extraction begins. Retrospective schema changes are a complete systematic failure that requires re-processing all previously collected data.

2. YOU MUST adopt the POLE schema as the standard framework: **Person, Object, Location, Event**. Every entity extracted during collection MUST resolve into one of these four canonical types.

3. You are REQUIRED to define all field names and data types for each POLE entity type before collection begins. Fields MUST be defined at this stage — ad hoc field creation during collection is FORBIDDEN.

4. ALWAYS specify the unique identifier fields for each entity type (e.g., passport number for Person, company registration number for Object). It is CRITICAL that deterministic matching rules are defined for these identifiers before any entity resolution begins.

5. YOU MUST record the finalised ontology in `research/investigation-strategy.md`. Any ontology change during the investigation MUST be logged as a Policy Deviation with full impact assessment on previously collected data.
