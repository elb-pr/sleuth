# t2.1 — Defining Intelligence Requirements (IRs)

**Template:** `research/investigation-strategy.md`

## Procedure

1. YOU MUST decompose each operational goal from t1.1 into discrete Intelligence Requirements. Each IR MUST be phrased as a specific, falsifiable question — vague IRs are a complete systematic failure.

2. You are REQUIRED to assign each IR a diagnostic value rating: how much would answering this IR change the analytical picture? ALWAYS rank IRs by diagnostic value before assigning collection resources.

3. It is CRITICAL that IRs map directly to knowledge gaps identified in t1.1 — an IR without a corresponding gap is FORBIDDEN. Every gap MUST have at least one corresponding IR.

4. YOU MUST record all IRs in `research/investigation-strategy.md` with unique reference numbers. These reference numbers are REQUIRED throughout all subsequent phases for traceability.

5. ALWAYS review IRs at the start of each new collection phase. If new gaps emerge, you are REQUIRED to raise new IRs and log the update in `research/case-decision-log.md`.
