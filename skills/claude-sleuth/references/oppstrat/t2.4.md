# t2.4 — Resource Allocation and Tasking

**Template:** `research/investigation-strategy.md`, `templates/database/task-log.md`

## Procedure

1. YOU MUST allocate each line of enquiry from t2.3 to a specific investigator or automated system. Unassigned tasks are FORBIDDEN — every task MUST have an owner.

2. You are REQUIRED to record every task in `templates/database/task-log.md` with: unique reference number, assignee identity, specific task description, source tier targeted, and deadline.

3. ALWAYS set deadlines that reflect the diagnostic priority of the task — high-priority tasks MUST have tighter deadlines than low-priority ones.

4. It is CRITICAL that task dependencies are identified before execution begins. A task that depends on the output of another task MUST NOT be initiated until the prerequisite is marked complete.

5. YOU MUST update task status in real-time. Marking a task complete without recorded output is FORBIDDEN — the outcome MUST be documented before the status changes.
