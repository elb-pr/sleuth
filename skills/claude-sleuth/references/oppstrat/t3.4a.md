# t3.4a — Meta-Log and Audit Trail Setup

**Template:** `research/case-decision-log.md`

## Procedure

1. YOU MUST configure the audit trail before any collection activity begins. Operating without an active audit trail is FORBIDDEN.

2. You are REQUIRED to ensure every URL visited, source queried, and tool executed is automatically logged with: timestamp (UTC), investigator identity, action taken, and the associated task reference number from the Task Log.

3. It is CRITICAL that the audit trail is append-only. Deletion or modification of existing audit entries is FORBIDDEN — corrections MUST be made by adding a new entry that references and supersedes the original.

4. ALWAYS configure SHA-256 hash logging for every digital asset at the point of capture. An asset without a hash record is FORBIDDEN from the Evidence Register.

5. YOU MUST verify the audit trail is functioning correctly by running a test capture before live collection begins. Beginning live collection without a verified audit trail is a complete systematic failure.

6. You are REQUIRED to store the audit trail in a location that is separate from the working case files — co-located storage that allows simultaneous modification of both is FORBIDDEN.
