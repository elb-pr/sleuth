# t13.3 — Applying Standardised Probability Language (ICD 203)

**Template:** `working/briefing.md`

## Procedure

1. YOU MUST map every probabilistic claim in the report to one of seven ICD 203 tiers:

   | Term | Probability Range |
   |---|---|
   | Almost certain / Nearly certain | 95–99% |
   | Very likely / Highly probable | 80–95% |
   | Likely / Probable | 55–80% |
   | Roughly even chance / Roughly even odds | 45–55% |
   | Unlikely / Improbable | 20–45% |
   | Very unlikely / Highly improbable | 5–20% |
   | Remote / Almost no chance | 1–5% |

2. You are REQUIRED to apply these rules without exception:
   - **Single-tier per sentence:** Mixing terms from different tiers in the same sentence is FORBIDDEN.
   - **Confidence/Likelihood separation:** It is CRITICAL that confidence (High/Moderate/Low) and likelihood expressions NEVER appear in the same sentence.
   - **No non-standard terms:** "might," "good chance," "could be," "it is possible," "we believe," "we think" are ALL FORBIDDEN.
   - **No plain-language substitution:** If simplification is needed, YOU MUST state the standardised term first, then add the percentage range in parentheses.

3. ALWAYS apply ICD 203 language throughout the ENTIRE report — not just the BLUF:
   - Judgments section (t13.2)
   - Source Summary Statement (t13.4)
   - Findings Memo conclusions (t15.1)

4. In UK defence/law enforcement contexts, JDP 2-00 probability vocabulary may be used as an alternative framework. It is CRITICAL that the two frameworks are NEVER interchanged within the same report.

5. YOU MUST conduct a pre-finalisation term audit: review every sentence expressing probability or uncertainty. Confirm ICD 203 compliance. Verify confidence/likelihood are not blended. Verify BLUF probability expressions are mathematically justified by the ACH inconsistency scores (t12.3). Submitting a report that fails this audit is FORBIDDEN.
