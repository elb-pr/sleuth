# t15.2 — Social Media Attribution (Account Attribution Report)

**Template:** `working/report.md`

## Procedure

1. YOU MUST only issue the Account Attribution Report when the investigation has produced evidence bearing on digital identity. It MUST be anchored to a specific testable hypothesis: **"Account X belongs to Individual Y."**

2. It is CRITICAL that you apply the evidence threshold BEFORE issuing:
   - Attribution conclusions REQUIRE at least one piece of independently verifiable evidence. Stylistic similarity or topical overlap alone is INSUFFICIENT and MUST NOT be used as the sole basis for attribution.
   - ALWAYS preserve all supporting digital evidence per t5.1 (SHA-256 hashing) and t5.2 (metadata/archive). Unarchived screenshots are INADMISSIBLE.

3. YOU MUST follow this standard structure — omitting any section is FORBIDDEN:

   **Executive Summary:** Platform, handle, attributed identity (if determined), attribution confidence level (ICD 203), one-paragraph evidence chain summary. It is CRITICAL that confidence and likelihood are NEVER blended in the same sentence.

   **Account Profile:** Platform, handle, display name, profile photograph, biography, account creation date (if recoverable), follower/following counts, posting frequency, date range of archived material. Each data point MUST carry a timestamp and archive reference.

   **Content Analysis:** Topics/narrative themes, geographic references, language and writing style (vocabulary, syntax, colloquialisms), temporal posting patterns (time zone indicators), self-referential disclosures.

   **Attribution Evidence** (YOU MUST present in descending reliability order):
   1. Unique identifiers: email/phone from registration, biographical data matching government records.
   2. Cross-platform matching: same handle, photo, writing style on platforms with stronger identity signals.
   3. Network connections: follower/following overlap with verified associates (indirect — corroboration only, NEVER primary identifier).
   4. Content-based corroboration: posts referencing events, places, knowledge only the attributed individual could possess.
   5. Technical evidence: EXIF metadata (where not platform-stripped), legally obtained IP indicators, device fingerprinting.

   **Counter-Attribution Analysis:** YOU MUST address: could the account be operated by multiple individuals? A sockpuppet? Could similarities be coincidental? ALWAYS apply Analysis of Alternatives logic from Phase 5.

   **Confidence Assessment** — YOU MUST classify as one of:
   - Confirmed: definitively linked via unique, verifiable identifier.
   - Probable: multiple independent evidence types consistent, no material contradictory evidence.
   - Possible: limited evidence, insufficient corroboration for a confident conclusion.
   - Uncertain: evidence suggests a connection but does not support positive attribution at any confidence level.

   **Operational and Legal Considerations:** Handling restrictions, platform ToS constraints, GDPR/privacy law considerations.

   **Source Documentation:** You are REQUIRED to list all archived evidence items with hash values, archival URLs, collection dates, cross-referenced to the Evidence Register.

4. ALWAYS apply these quality standards:
   - Verification methodology (reverse image search, geolocation, chronolocation, metadata analysis) MUST be stated for each evidence item.
   - Inference stacking is FORBIDDEN: each step in the attribution chain MUST trace to a discrete, sourced evidence item.
