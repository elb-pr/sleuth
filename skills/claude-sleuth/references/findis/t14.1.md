# t14.1 — Finalising the Case Summary Record

**Template:** `research/case-decision-log.md`

## Procedure

1. YOU MUST update the Case Summary Record (initiated in t3.1a) to reflect final status:
   - **Case Status:** Active → Completed / Closed (with reasons) / Referred (to another agency).
   - **Completion Date:** Date analytical conclusions were finalised and briefing issued.
   - **Duration Record:** Elapsed time from opening (t3.1a) to completion.

2. You are REQUIRED to finalise the Scope Statement to reflect what was ACTUALLY investigated — which may differ from the original ToR due to Policy Deviations logged in t4.3:
   - ALWAYS list all IRs from the ICP (t2.1) with status: Fully Addressed / Partially Addressed / Not Addressed (with reasons).
   - YOU MUST document any out-of-scope findings and how they were handled (referred, flagged, or noted for potential future action). Silent omission of out-of-scope findings is FORBIDDEN.

3. It is CRITICAL that you confirm the final personnel and authorisation record:
   - Lead investigator identity.
   - Full team composition including any changes from the original.
   - Supervisory sign-off: reviewer identity and date of final review (t3.3a protocol). A case file without supervisory sign-off is FORBIDDEN.

4. YOU MUST compile the Document Register — the complete case file index:
   - Intelligence Collection Plan, Task Log, Evidence Register, Source Register, Entity Register, Chronological Matrix, Network Analysis outputs, ACH Matrix, Analytical Briefing.
   - Each item MUST include: storage location and access restrictions.
   - You are REQUIRED to include the Evidence Vault reference (t14.3) with confirmation of final hash verification.
   - ALWAYS include the Dissemination Log reference (t14.2).
