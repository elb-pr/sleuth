# t6.2 — Parsing Unstructured Text

**Tool:** `entity_extractor.py`

## Procedure

1. YOU MUST classify incoming data before selecting an extraction strategy:
   - **Structured:** Tabular CSV or database exports — apply field mapping to align source column headers to POLE schema keys.
   - **Semi-structured:** Web pages with predictable layouts, social media profiles, PDF forms — apply template-based extraction targeting predictable field locations.
   - **Unstructured:** Free-text narrative (witness statements, intercepted communications, news articles) — apply Named Entity Recognition (NER).

2. For unstructured inputs, YOU MUST run NER to extract at minimum: person names, organisation names, locations, dates/times, monetary values, identification numbers (passport, company registration, phone, email), and vehicle identifiers. Use spaCy, Hugging Face Transformers, or Stanza. For multi-language investigations, multilingual NER models are REQUIRED.

3. It is CRITICAL that you apply coreference resolution to link multiple references to the same entity ("John Smith", "Mr Smith", "the defendant", "he") before creating records. Skipping coreference resolution is FORBIDDEN — it produces duplicate entries that corrupt the Entity Register.

4. YOU MUST map every extracted entity to the POLE schema. Missing fields MUST be left explicitly blank — inferring or guessing values is FORBIDDEN. Relationship phrases ("director of", "employed by", "transferred funds to") MUST generate typed link records using the vocabulary defined in t6.1.

5. ALWAYS normalise all date/time references to UTC and attach as `dateObserved` or `dateRange` metadata on the relevant record. Non-UTC timestamps are FORBIDDEN in the Entity Register.

6. You are REQUIRED to attach source provenance to every extracted record: Source Register identifier (t5.3), passage location (page/paragraph/line), and extraction method (manual / automated / hybrid). Automated extractions MUST be flagged for human review before being committed to the Entity Register.

7. ALWAYS apply confidence tagging: High (unambiguous), Medium (requires corroboration), Low (ambiguous). It is CRITICAL that low-confidence extractions are quarantined for review — ingesting them directly is FORBIDDEN. YOU MUST run a deduplication check against the Entity Register (t6.3) before committing any new record.
