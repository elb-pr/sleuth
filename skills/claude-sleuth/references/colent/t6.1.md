# t6.1 — Implementing the POLE Schema

**Template:** CSDb entity schema (`add_entity` tool), `research/investigation-strategy.md`

## Procedure

1. It is CRITICAL that you define mandatory fields for every entity type before any data extraction begins. Schema changes after extraction starts are FORBIDDEN — YOU MUST freeze the schema first.

   - **Person:** `givenName`, `familyName`, `birthDate`, `nationality`. Optional: aliases, address history, employment, phone, email, social handles, ID documents.
   - **Object:** Vehicles (VIN, plate, make, model), financial accounts (type, identifier, platform), devices (IMEI, number), documents (type, hash, archive URL), cryptocurrency wallet addresses.
   - **Location:** Address or coordinates (mandatory), type classification (residential / commercial / institutional). Optional: grid reference, ownership/tenancy records.
   - **Event:** Type, UTC-normalised date/time, location reference. Optional: participant list, description, associated entity references.

2. YOU MUST define the relationship vocabulary before extraction begins. Keep link types minimal: `ASSOCIATED_WITH`, `OWNS`, `EMPLOYS`, `COMMUNICATES_WITH`, `FAMILY_OF`, `DIRECTOR_OF`, `LOCATED_AT`, `PARTY_TO`, `WITNESSED_BY`. Every asymmetric relationship MUST be directional.

3. ALWAYS attach metadata to every relationship link: source attribution, date range active, confidence indicator, analyst notes. Bespoke ad-hoc link types are a documented failure mode — YOU MUST standardise to the controlled vocabulary. Deviating from the controlled vocabulary is FORBIDDEN.

4. You are REQUIRED to reserve extensibility fields (`Notes`, `customProperties`) for data that does not fit predefined structure. Redesigning the schema mid-investigation is FORBIDDEN.

5. It is CRITICAL that schema sign-off is obtained from the lead investigator or supervisor before t6.2 parsing begins. YOU MUST verify downstream compatibility:
   - The schema MUST produce UTC-normalised Event records for the Chronological Matrix (t9.1).
   - The schema MUST produce typed edges for the Network Architecture Template (t10.1).

6. YOU MUST map each IR from `research/investigation-strategy.md` to specific entity types and fields. If a data point does not populate any field relevant to any IR, do not extract it — orphan records degrade database utility without advancing the investigation.
