# t9.1 — Master Chronology Construction

**Template:** `analysis/chronological-matrix.md`

## Procedure

1. YOU MUST conduct a systematic source sweep: extract all Event-type entities from the Entity Register, all timestamped entries from `templates/database/task-log.md`, and all temporal data from Subject and Corporate Profiles (t8.1). Excluding any category of dated evidence is FORBIDDEN.

2. You are REQUIRED to include the following fields on every entry in the master chronology:
   - Date/Time (UTC-normalised)
   - Event Description (factual statement only — analytical interpretation is FORBIDDEN at this stage)
   - Entity References (which POLE entities are involved)
   - Source Reference (Source Register identifier)
   - Source Confidence (Admiralty 1–6 from t3.1b)
   - Temporal Certainty (Exact / Approximate / Estimated / Conflicting)

3. It is CRITICAL that ALL timestamps are normalised to UTC before sequencing. YOU MUST record the original timestamp, timezone offset applied, and UTC result for every conversion. For timestamps without a timezone, document the assumption used. DST transitions MUST be verified — system logs that do not adjust for DST MUST be corrected manually.

4. ALWAYS treat first-pass assembly as purely mechanical — sort all events by timestamp with no analytical filtering. YOU MUST include all dated events regardless of apparent relevance. Cherry-picking at this stage is FORBIDDEN.

5. YOU MUST detect duplicates: events appearing in multiple sources are REQUIRED to be recorded once, with all contributing sources cited. If sources provide different timestamps for the same event, YOU MUST flag as a conflict for t9.3.

6. You are REQUIRED to format the output as a structured matrix in `analysis/chronological-matrix.md` with cross-reference links to Entity Register records, Evidence Register items, and Source Register entries. This matrix is the primary temporal reference for the ACH Matrix (t11.x) and the Final Briefing (t13.x) — an incomplete or unlinked matrix is a complete systematic failure.
