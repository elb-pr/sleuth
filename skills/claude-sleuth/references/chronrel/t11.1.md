# t11.1 — Content Verification (Deep Analytical Pass)

**Tool:** `content_archiver.py`, `geolocation.py`

Where t3.3b performed initial vetting to grade incoming intelligence, this task deploys the **Content Verification Report** as a structured analytical product that authenticates the provenance of key digital assets and grounds them in the investigation's spatial and temporal framework.

## Procedure

### 1. Asset Selection
Not every digital asset warrants full verification. YOU MUST prioritise assets based on investigative significance:
- **Diagnostic Assets:** Media that, if authenticated, would significantly alter the ACH matrix — these are REQUIRED to undergo full verification.
- **Disputed Assets:** Media whose provenance has been challenged, or which originated from low-reliability sources (Admiralty grade "F") — REQUIRED for full verification.
- **High-Impact Assets:** Media to be presented in the final briefing or submitted as legal evidence — YOU MUST apply the highest verification standard. Submitting unverified high-impact assets as evidence is FORBIDDEN.

### 2. Multi-Step Authentication
Each selected asset MUST undergo the full structured verification pipeline — skipping any step is FORBIDDEN:
- **Reverse Image/Video Search:** YOU MUST submit to reverse search engines (Google Reverse Image Search, TinEye, Yandex Images). If the media predates the claimed context, it has been repurposed and MUST be flagged.
- **Geolocation:** YOU MUST match visual cues against satellite imagery and street-level photography. You are REQUIRED to record coordinates, visual cues used, and confidence level. Unverified geolocation claims are FORBIDDEN in analytical products.
- **Chronolocation:** YOU MUST independently estimate capture time through shadow analysis (SunCalc/pysolar), weather cross-referencing (Open-Meteo, Weather Underground), vegetation analysis, and visible temporal markers. ALWAYS record the methods used.
- **Metadata Verification:** YOU MUST extract and compare EXIF data against geolocation and chronolocation findings. Inconsistencies MUST be flagged immediately.

### 3. Manipulation Detection
For high-stakes assets, YOU MUST assess whether the media has been digitally altered:
- **Error Level Analysis (ELA):** Detects composite imagery through compression artefact inconsistencies. You are REQUIRED to run ELA on all high-impact assets.
- **Cloning Detection:** Identifies duplicated regions indicating objects have been added or removed.
- **Metadata Inconsistencies:** Camera model inconsistent with resolution/format, or editing software signatures, MUST be flagged as indicators of post-capture manipulation.

### 4. Content Verification Report Output
YOU MUST compile results into a structured **Content Verification Report** that becomes part of the permanent case file:
- **Verification Status:** ALWAYS classify as: Verified, Partially Verified, Unverified, or Falsified. Omitting a classification is FORBIDDEN.
- **Spatial Coordinates:** Geolocated coordinates and confidence level.
- **Temporal Estimate:** Chronolocated date/time range and derivation methods.
- **Admiralty Grade Update:** You are REQUIRED to feed the verification outcome back to the Source Register — it potentially upgrades the source's credibility rating.
- **Evidentiary Linkage:** The report MUST be linked to the corresponding Evidence Register entry (hash-verified via t5.1) and to any Entity Register records or master chronology entries that reference the asset.
