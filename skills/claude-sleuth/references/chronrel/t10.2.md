# t10.2 — Quantitative Social Network Analysis (SNA)

**Tool:** `network_analyzer.py`

## Procedure

1. It is CRITICAL that you run SNA on the edge list (t10.1) BEFORE visual interpretation. Objective metrics MUST precede visual layout — visual impression is susceptible to layout bias and the hairball effect. Proceeding to t10.3 before completing this task is FORBIDDEN.

2. YOU MUST compute the following metrics for every node:

   - **Degree Centrality:** Count of direct connections. In directed graphs, YOU MUST compute separately as in-degree and out-degree. High-degree nodes are the most connected entities — potential leadership figures, key facilitators, or communication hubs.

   - **Betweenness Centrality:** Frequency with which a node lies on the shortest path between other node pairs. High-betweenness nodes are structural chokepoints — ALWAYS flag these as the most strategically important targets. A mid-level facilitator with low degree but high betweenness controls all communication between clusters.

   - **Closeness Centrality:** Average shortest path length to all other nodes. High-closeness nodes disseminate or receive information most efficiently.

3. YOU MUST run community detection (Louvain modularity or Girvan-Newman) to partition the network into clusters. You are REQUIRED to identify inter-cluster bridge nodes by cross-referencing with betweenness centrality results. Nodes appearing as both high-betweenness AND inter-cluster bridges MUST be flagged as the network's critical structural vulnerabilities.

4. If the edge list includes date ranges, YOU MUST run SNA across time slices to show network evolution — clusters forming, dissolving, or merging.

5. You are REQUIRED to output a metrics table listing every entity with its centrality scores, cluster membership, and structural role (hub / bridge / peripheral). This table MUST accompany the visual output from t10.3.

6. Recommended tooling: **NetworkX** (Python, all centrality measures and community detection), **Gephi** (large-scale visual SNA), **IBM i2 Analyst's Notebook** (law enforcement contexts).
