# t10.1 — Edge List Generation

**Template:** `analysis/network-architecture.md`

## Procedure

1. YOU MUST systematically export all typed relationship links from the Entity Register. It is CRITICAL that you filter out any link lacking source attribution before export — unsourced edges are FORBIDDEN in the edge list.

2. You are REQUIRED to include the following four mandatory fields on every edge:
   - **Source Entity:** Unique Entity Register identifier of the acting entity.
   - **Target Entity:** Unique Entity Register identifier of the receiving entity.
   - **Relationship Type:** From the controlled vocabulary defined in t6.1. Free-text or ad-hoc labels are FORBIDDEN.
   - **Evidence Citation:** Source Register identifier (t5.3) documenting this relationship.

3. ALWAYS add the following supplementary metadata to every edge:
   - **Confidence Level:** Admiralty credibility (1–6) — governs visual grammar in t10.3.
   - **Date Range:** Start and end dates when the relationship was active.
   - **Weight/Intensity:** Transaction volumes, communication frequency, or other quantifiable measures where available.
   - **Directionality:** Unidirectional or bidirectional.

4. YOU MUST deduplicate: if the same relationship appears in multiple sources, you are REQUIRED to record it as a single edge with all contributing sources cited and the highest corroboration status applied.

5. ALWAYS exclude inferred relationships from the primary edge list — reserve them for Two-Mode Analysis (t10.4). Including inferred relationships in the primary edge list is FORBIDDEN.

6. It is CRITICAL that you run the following quality gate before proceeding to t10.2. It is FORBIDDEN to proceed with a failing quality gate:
   - Every edge has a populated Evidence Citation.
   - Every Source and Target entity resolves to an Entity Register record.
   - Every Relationship Type matches the controlled vocabulary.

7. You are REQUIRED to output in tabular format (CSV/Markdown) for NetworkX/Gephi import, and as Mermaid.js graph syntax for text-based environments.
