# t10.3 — Visual Grammar Application

**Tool:** `network_analyzer.py`

## Procedure

1. YOU MUST apply the following node visual grammar:
   - **Shape/Icon:** Distinct representation per entity type (person silhouette, building, map pin, calendar, vehicle, device).
   - **Size:** Scale by degree centrality or transaction volume.
   - **Colour:** Encode cluster membership (from t10.2), entity type, or risk status (red = sanctioned, amber = PEP, green = cleared).

2. You are REQUIRED to apply the following edge visual grammar. It is CRITICAL that the confidence distinction is applied without exception:
   - **Solid line:** Admiralty credibility 1–2 — confirmed or probably true. The investigation is prepared to defend this relationship.
   - **Dashed/dotted line:** Admiralty credibility 3 or lower, or inferred/unconfirmed relationships. Communicates epistemic doubt directly to the viewer.
   - **Line weight:** Encodes relationship intensity (transaction volume, communication frequency).
   - **Directional arrows:** REQUIRED for all asymmetric relationships (fund flows, employment).
   - **Edge labels:** Relationship type from the controlled vocabulary (t6.1).

3. ALWAYS include a legend on every visualisation explaining: node shapes, node colours, line styles (solid = confirmed, dashed = unconfirmed), line weight scale, arrow direction meaning, and temporal scope of the data. A visualisation without a legend is FORBIDDEN.

4. YOU MUST apply hairball prevention:
   - Filter to display only edges above a minimum confidence or weight threshold. The full dataset MUST remain in the edge list — the visualisation is a curated analytical view.
   - Segment complex networks into sub-charts by cluster, entity type, time period, or geographic region.
   - ALWAYS document the layout algorithm used (hierarchical, force-directed, circular) — different layouts can make the same data appear structurally different.

5. You are REQUIRED to output in the following formats: interactive HTML (pyvis), static PNG/SVG for reports, Mermaid.js for text-based environments, GEXF/GraphML for further platform import.
